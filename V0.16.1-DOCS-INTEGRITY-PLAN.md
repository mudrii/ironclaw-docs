## Phase 3: Audit Methodology

### Each Subagent Must:

1. **Read Source Files**
   - All relevant `.rs` files in assigned domain
   - Check function signatures, struct definitions, constants
   - Note any recent changes in git history

2. **Read Documentation**
   - All relevant `.md` files in ironclaw-docs
   - Cross-reference with source

3. **Validate Facts**
   - Environment variable names
   - Configuration options
   - API endpoints
   - Feature flags
   - Default values
   - Version numbers
   - File paths

4. **Identify Discrepancies**
   - Missing features in docs
   - Outdated information
   - Incorrect defaults
   - Wrong version references
   - Deprecated features still documented
   - New features not documented

5. **Output Format**
   ```markdown
   ## Domain: [Name]
   
   ### Files Audited
   - Source: [list]
   - Documentation: [list]
   
   ### Findings
   
   #### [Category]: [Brief description]
   - **Location:** [file:line or section]
   - **Source Truth:** [what code actually does]
   - **Documentation:** [what docs say]
   - **Severity:** [Critical/High/Medium/Low]
   - **Action:** [Update docs/Update code/Investigate]
   
   ### Summary
   - Total issues: [N]
   - Critical: [N]
   - High: [N]
   - Medium: [N]
   - Low: [N]
   ```
